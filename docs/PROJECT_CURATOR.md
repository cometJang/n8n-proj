# AI 동네 큐레이터 (AI Neighborhood Curator)
## 프로젝트 최종 기획서 (Technical Refined v3.2)

**문서 버전**: v3.2 (Architectural Integrity)  
**최종 업데이트**: 2026-02-03  
**상태**: Phase 1 (MVP) 완료, Phase 2 (Automation) 진입 준비  

---

## 1. 프로젝트 개요
부동산 가격이라는 단편적 정보에서 벗어나, **통신 빅데이터 기반의 라이프스타일 지표**와 **부동산 실거래가**를 결합하여 사용자 페르소나별 최적의 동네를 추천하는 AI 에이전트 시스템입니다.

---

## 2. 문제 정의: 데이터의 홍수 속 '의사결정의 마비'

### 2.1 주요 Pain Point: 파편화된 정보와 인지적 노동
현재 부동산 시장의 문제는 '정보의 부족'이 아니라 **'파편화된 정보로 인한 인지적 부하(Cognitive Load)'**입니다.
- **의사결정 피로도(Decision Fatigue)**: 매물 가격, 출퇴근 시간, 동네 분위기를 각기 다른 플랫폼에서 확인하고 수동으로 조합해야 하는 과정에서 과도한 인지적 노동이 발생합니다.
- **주거 실패의 사회적 비용**: 불완전한 정보로 인한 잘못된 주거 선택은 개인에게 연간 **약 240시간 이상의 출퇴근 시간 낭비**와 재이사 비용 등 막대한 경제적 기회비용을 발생시킵니다.
- **데이터 격차(Data Gap)**: 고차원의 통신 빅데이터가 전문가의 영역에만 머물러 있어, 개인의 라이프스타일 의사결정에 민주적으로 활용되지 못하고 있습니다.

---

## 3. 핵심 설계 철학: 데이터 헌법 (Data Constitution)
본 프로젝트는 데이터 오남용을 방지하고 분석의 신뢰성을 확보하기 위해 다음 원칙을 준수합니다.

1. **계층화된 설계 (Layered Architecture)**:
    - **결정 레이어 (Decision Layer)**: 순위 결정은 오직 검증된 통신/부동산 수치 데이터만 사용.
    - **설명 레이어 (Explanation Layer)**: 정성적 해석(관심집단 등)은 순위 확정 후 리포트의 풍부함을 위해서만 사용.
2. **헌법적 방화벽 (Constitutional Firewall)**: 설명용 데이터는 절대 점수 산정에 영향을 줄 수 없음.
3. **명명 표준 (Naming Convention)**: 내부 데이터 컬럼은 `한글_언더바`, 소스 코드는 `English_Standard`를 적용하여 가독성과 전문성 동시 확보.

## 4. 시스템 아키텍처 (Technical Structure)

### 4.1 모듈별 역할
| 모듈 | 역할 | 핵심 기능 |
| :--- | :--- | :--- |
| `main.py` | 지휘자 (Orchestrator) | 전체 분석 파이프라인 제어 및 페르소나 주입 |
| `data_engine` | 데이터 정제소 | `smart_data_load` (캐싱), `standardize` (한글화/집계) |
| `scoring` | 결정 엔진 | `calculate_scores` (백분위 기반 정적 랭킹) |
| `reporting` | 리포트 & 시각화 | `build_reports` (JSON/CSV), `create_radar_chart` (HTML) |
| `explainer` | 설명 엔진 | `extract_signals` (관심사 기반 라이프스타일 해설) |

## 5. 타겟 페르소나 및 가중치 전략
사용자의 성향에 따라 `policies/personas.json`을 통해 동적으로 가중치가 적용됩니다.

1. **💼 직장 노비형 (`career_slave`)**: 출퇴근(50%) > 월세(20%) > 인프라(20%)
2. **☕ 생활 만족러 (`life_satisfier`)**: 분위기(40%) > 인프라(30%) > 안전(10%)
3. **💰 가성비 헌터 (`value_hunter`)**: 월세(60%) > 인프라(10%) > 분위기(10%)
4. **🏠 혼자 잘살형 (`solo_liver`)**: 안전(50%) > 분위기(20%) > 인프라(10%)
5. **💻 디지털 노마드 (`digital_nomad`)**: 분위기(30%) > 인프라(30%) > 안전(20%)

## 6. 데이터 파이프라인

### 6.1 데이터 소스
- **결정용**: 실거래가 CSV, 통신정보 XLSX (총인구, 1인가구, 배달사용일수 등)
- **설명용**: 10개 관심집단수 XLSX (문화, 자기계발, 쇼핑 등)
- **교통용**: Google Maps API 사전 계산 캐시

### 6.2 데이터 정규화 알고리즘
- 절대값 비교가 아닌 **백분위(Rank Percentile)** 방식을 채택하여 지역 간 규모 편차에 의한 오류를 원천 차단.

---

## 7. 최종 산출물 (Outputs)
1. **`report.json`**: n8n 및 시스템 연동을 위한 규격화된 데이터 리포트.
2. **`top3_final.csv`**: 사용자가 즉시 확인 가능한 한글 컬럼 기반 요약 파일.
3. **`radar_final.html`**: 지역별 강점을 직관적으로 비교하는 프리미엄 레이더 차트.
